ShoppingGate Flutter SDK

Android Native Integration Guide (AAR)

Complete technical guide for embedding the ShoppingGate SDK into native Android apps using Jetpack Compose.

1. Prerequisites

Verification of the following environment settings is required for a successful build:

  • JDK: Version 17 Strict Requirement
  • Android Studio: Ladybug (2024.2.1) or higher
  • Min SDK: API 29 (Android 10)

2. Repository Setup

The SDK is delivered as a local Maven repository. Place the ShoppingGateSDK folder in your project's root directory.

Root-Directory/
├── app/
├── ShoppingGateSDK/ <-- Local SDK folder
├── build.gradle.kts
└── settings.gradle.kts

3. Host Configuration

A. Settings Gradle

settings.gradle.kts
dependencyResolutionManagement {
    repositoriesMode.set(RepositoriesMode.FAIL_ON_PROJECT_REPOS)
    repositories {
        google()
        mavenCentral()
        
        // Local ShoppingGate SDK
        maven {
            url = uri("${rootProject.projectDir}/ShoppingGateSDK")
        }
        
        // Flutter engine binaries
        maven {
            url = uri("https://storage.googleapis.com/download.flutter.io")
        }
    }
}

B. App-Level Gradle (build.gradle.kts)

Ensure multiDexEnabled and abiFilters are correctly set.

app/build.gradle.kts
android {
    defaultConfig {
        ndk {
            abiFilters += listOf("armeabi-v7a", "arm64-v8a", "x86_64")
        }
        multiDexEnabled = true
    }
    
    compileOptions {
        isCoreLibraryDesugaringEnabled = true 
        sourceCompatibility = JavaVersion.VERSION_17
        targetCompatibility = JavaVersion.VERSION_17
    }

    kotlinOptions {
        jvmTarget = "17"
    }
}

dependencies {
    implementation("com.shoppinggate.sdk.customer:flutter_release:1.0")
    coreLibraryDesugaring("com.android.tools:desugar_jdk_libs:2.1.5")
}

C. Kotlin Compiler Task

build.gradle.kts (Root)
import org.jetbrains.kotlin.gradle.tasks.KotlinJvmCompile

tasks.withType<KotlinJvmCompile>().configureEach {
    compilerOptions {
        jvmTarget.set(org.jetbrains.kotlin.gradle.dsl.JvmTarget.JVM_17)
    }
}

4. Android Manifest Registry

Note the usage of Theme.ShoppingGateFlutterSDK and storage permissions.

AndroidManifest.xml
<manifest xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:tools="http://schemas.android.com/tools">

    <uses-permission android:name="android.permission.ACCESS_FINE_LOCATION"/>
    <uses-permission android:name="android.permission.INTERNET"/>
    
    <!-- Storage for older Android versions -->
    <uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE"
        tools:ignore="ScopedStorage" />
    <uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE"
        tools:ignore="ScopedStorage" />

    <application>
        <activity
            android:name="io.flutter.embedding.android.FlutterActivity"
            android:theme="@style/Theme.ShoppingGateFlutterSDK"
            android:configChanges="orientation|keyboardHidden|keyboard|screenSize|locale|layoutDirection|fontScale|screenLayout|density|uiMode"
            android:hardwareAccelerated="true"
            android:windowSoftInputMode="adjustResize" />
    </application>
</manifest>

5. Launching the SDK

MainActivity.kt
import io.flutter.embedding.android.FlutterActivity

class MainActivity : ComponentActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContent {
            Button(onClick = {
                startActivity(FlutterActivity.createDefaultIntent(this))
            }) {
                Text("Launch ShoppingGate")
            }
        }
    }
}

6. Troubleshooting

  • Hot Reload: Not supported for AAR integration. Clean and Rebuild to see changes.
  • ABI Issues: Ensure your emulator/device matches the abiFilters.
  • Clean Build: Use ./gradlew clean if classes are not found.
